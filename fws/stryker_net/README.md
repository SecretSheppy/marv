# Stryker MTE Frameworks

This document details how to use any of the mutation testing frameworks that Marv supports via the Stryker MTE schema.

## Frameworks

The full list frameworks supported through their MTE schema output. To use these frameworks in Marv you must configure
them to export results in the Stryker MTE schema format. 

> [!IMPORTANT]
> These are not generic framework implementations and
so cannot be used in place of each other, i.e., if you have output from infection but try to run it through Marv with
the `mull` framework implementation then Marv will go wrong. This is because Marv generates and repairs framework
specific data where it is not provided or is broken.

* [Mull](https://mull-project.com/)
* [stryker&#8288;&#45;&#8288;net](https://github.com/stryker-mutator/stryker-net)
* [stryker&#8288;&#45;&#8288;js](https://github.com/stryker-mutator/stryker-js)
* [infection](https://github.com/infection/infection)
* [strkyer4s](https://github.com/stryker-mutator/stryker4s)

## Getting Started

1. Run the `marv init` command for your framework of choice. This generates the required `.marv.yml` configuration file.
In the below example we configure the infection framework. All you have to do to change the framework is replace the
framework name with the one you are configuring.

```terminaloutput
marv init -f infection
```

2. Edit the `mte-json` field under the name of the framework you are configuring inside the `.marv.yml` file. An example
is described in the below YAML file.

```yaml
# Enable the framework of choice, in this case infection.
infection:
    # The mte-json field is available for all frameworks listed in this document.
    # This field should contain the path to the JSON generated by the mutation
    # testing framework.
    mte-json: report.json
```

3. Run the `marv` command to launch Marv and click the localhost URL to open the Marv interface.

```terminaloutput
marv
```
